# План реализации «Барбер и Барби»

## Цель

Собрать mobile-first одностраничный сайт на Next.js + TypeScript, готовый к Vercel, с плакатным hero по утверждённой концепции B, реальными фотографиями салона, безопасной заявкой в Telegram, юридическими документами-шаблонами и локальным SEO-фундаментом.

## Этапы

1. Создать минимальный Next.js App Router scaffold, базовые токены CSS и тестовую инфраструктуру.
2. До production-кода написать unit-тесты валидации/нормализации заявки.
3. Реализовать страницу: hero, услуги, реальные фото, преимущества, отзывы с раскрытием, контакты, форма, sticky CTA.
4. Реализовать `app/api/booking/route.ts`: валидация, honeypot, ограничение частоты, Telegram Bot API, понятные статусы.
5. Добавить `/privacy` и `/consent`, placeholders реквизитов оператора, чекбокс согласия, SEO metadata, JSON-LD BeautySalon, OG, favicon, robots и sitemap.
6. Скопировать утверждённые фото в `public/images`, оптимизировать размеры/форматы и проверить alt-тексты.
7. Запустить типы, линтер, тесты, production build и smoke-сценарии формы/документов.

## Проверки готовности

- `npm run typecheck`, `npm run lint`, `npm test`, `npm run build` проходят.
- Без согласия форма не отправляется; дата/время отображаются как пожелание.
- Секреты Telegram отсутствуют в клиенте, git и `.env.example`.
- NAP одинаков в видимом контенте и JSON-LD; юридические placeholders явно отмечены.
